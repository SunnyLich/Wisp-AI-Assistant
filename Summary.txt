Wisp — Summary
Product concept
A small persistent icon on the desktop, not a window. Invoked by hotkey, delivers a short verbal answer plus inline text, then disappears. Click the icon for the full text reply. Cloud-based LLM. Windows-first (Raycast is Mac-only — real gap). Utility-first positioning, not companion-first.
Target user
General computer users, not devs. People who get errors, see things they don't understand, want quick explanations without context-switching to ChatGPT.
Frictions targeted

Opening a window → small persistent icon, hotkey invocation, no app to launch
Typing the question → highlight + hotkey auto-captures the selection or screen snippet, no typing needed
Reading walls of text → voice answer for the short version, text expansion on demand

Key UX moves

Intent shortcuts. ctrl+u + arrow keys map to common query types what is this, how do I fix this, etc. Like vim verbs for AI queries — skips prompt-writing for the most common cases.
Input modalities. Highlighted text and screen snippets (Win+Shift+S style) as primary inputs.
Filler audio. Short canned phrases (hm..., let me think..., about that...) play immediately on hotkey to mask LLMTTS latency and add personality. Must be under 1s and overlap with real response generating in the background — otherwise it adds time instead of hiding it.
Visual feedback on the icon. The icon animates the moment the hotkey is pressed (eyes look up, bounce, etc.). Perceived latency is mostly about did the system register me, not raw response time.
Personality toggle. Cutepersonality mode opt-in; utility default for the hate-it crowd.

Technical decisions

No local model — too slow for the quality needed.
Streaming end-to-end. Cloud LLM with fast TTFT (Groq Llama, Claude, or GPT) plus low-latency streaming TTS (Cartesia ~75ms, ElevenLabs ~300-500ms).
Pre-cached filler audio as local wav files. Plays instantly on hotkey for sub-50ms acoustic feedback. Randomize from 5-10 variants so it doesn't grate.
Latency budget. 1.5-2s end-to-end to first real audio is the target. 3s is the ceiling — above that the friction is back.

First build priority
Prototype the latency loop before any UI hotkey → screen capture → LLM → TTS → audio out. Measure actual numbers on the chosen stack. If you can't hit under 3s with no UI in the way, the product premise needs revisiting before sinking weeks into the icon. If you can hit 1.5-2s, the rest is execution.
Strategic positioning
Differentiation against Copilot rests on model quality (ClaudeGPT vs Copilot's reputation), Windows-first availability, and voice-first short answers. Not a moat against Microsoft — they could license a better model tomorrow — but a real wedge while it works. Brand reputation (Copilot bad, this good) can outlast actual model parity.
Deferred

Pricing model — wait until the product works.
Monetization strategy — same.
General user distribution — real problem (general users don't install niche tools), but unsolvable in a doc. Needs shipping to learn.

Known risks worth tracking

Microsoft Copilot ships similar at OS level within 12-24 months. Timing matters.
The shortcut-memorization paradox general users won't memorize ctrl+u + arrow, power users don't need voice handholding. Bridge needed somewhere.
Privacy pushback on screen snippets going to an LLM. Have an answer ready.
Unit economics. Even before pricing, know cost-per-query on the chosen API. A free version at $0.03query × 30 queriesday × N users gets ugly fast.
